# RULES (프로젝트 작업 규칙)

> 이 문서는 AI가 이 프로젝트에서 작업할 때 지켜야 할 규칙을 정의합니다. `../AI Prompt Library`의 모드 프롬프트(Strict/Rapid/Reviewer)가 "일반적인 절차"를 정한다면, 이 문서는 **이 프로젝트만의 고유한 규칙**을 정합니다.

---

## 기본 작업 모드

- 기본 모드: **멘토 모드** — 사용자는 코딩 초보. 한 번에 하나의 단계만 진행하고,
  사용자가 완료를 확인하기 전에는 다음 단계로 넘어가지 않는다.
  쉬운 말로 설명하고, 변경한 파일과 이유를 항상 함께 알려준다.
- 예외: 사용자가 "알아서 쭉 진행해"라고 명시하면 여러 단계를 연속 진행 가능.

---

## 기술 스택

- 언어/프레임워크: 순수 HTML/CSS/JavaScript(ES5 스타일) — **프레임워크·빌드 도구 없음**
- 데이터베이스: 없음. 설정은 브라우저 localStorage(키 `hb_*`), 막히면 메모리 폴백
- 배포 환경: GitHub Pages (`main` 브랜치, push 시 자동 재배포)
- 데이터 출처: 한강홍수통제소(HRFCO) 오픈API — 방문자 브라우저가 직접 호출

---

## 코딩 규칙

- **단일 파일 유지**: 모든 코드는 index.html 하나에. 파일 분리·번들러 도입 금지(공유 단순화 목적).
- 코드 스타일: 기존 코드와 동일하게 — `var` 기반 ES5 스타일, 압축된 CSS 한 줄 규칙,
  한국어 UI 문구. 새 문법(let/const, 화살표 함수) 도입하지 않기(구형 모바일 브라우저 호환).
- 주석 규칙: WHY만 설명(예: "KST 환산 이유"). WHAT 나열 금지.
- 테스트 규칙: 자동화 테스트 없음. 대신 **로컬 미리보기로 실측 검증** 후 배포
  (`python -m http.server 8137`, `.claude/launch.json`의 preview 설정 사용).

---

## 금지 사항

- 승인 없이 프로덕션 배포하기 — 단, 사용자가 요청한 작업의 완료 커밋·push는 작업 흐름에 포함됨
- 검증되지 않은 안전 수치(base, noGo, warn) 임의 변경 — 반드시 근거(회사 시스템 값·공식 자료)와
  함께 사용자 확인 후 반영
- HRFCO API 키를 "숨겼다"고 주장하는 구조 도입 — 이 API는 해외 IP 차단이라 무료 프록시 불가.
  키 노출은 **의도된 결정**(무료·읽기 전용·재발급 가능)이므로 어설픈 은닉 시도로 복잡도 올리지 않기
- 폐기된 워커 방식(Cloudflare Worker/allorigins 등 해외 프록시) 재도입

---

## 승인이 필요한 작업

- 교량 추가/삭제, 임계값(noGo/warn)·기준높이(base) 변경
- UI 구조 변경(카드 레이아웃, 표시 문구 등 사용자가 보는 부분)
- 외부 리소스(CDN 폰트 등) 추가
- CONFIG 구조 변경

---

## 커밋 / 문서화 규칙

- 커밋 메시지 형식: `타입: 한국어 요약` (예: `feat: 데이터 신선도 경고 추가`,
  `docs: 배포 방식 갱신`). 본문은 선택, 한국어.
- 각 작업 후 갱신해야 하는 문서: `TASKS.md`, `CHANGELOG.md`, `AI_HANDOFF.md`, **`CLAUDE.md`**
  (CLAUDE.md 상단 "작업 규칙" 참고 — 문서 갱신을 같은 커밋/푸시에 포함)

## 원격 저장소 (GitHub 등)

- 사용 여부: **사용**
- 저장소 주소: https://github.com/OGISD/HanGangBUS (`origin`, `main` 추적)

  실험적·미완성 코드는 커밋하지 않는다(배포되어 버림).
